UI Kit

reapp-ui composes the bulk of re/app. It's a UI kit that gives you the pieces you need to build beautiful app UI's.

Themeing

Powerful themeing system. Themes are composed of three layers: constants, styles, animations. Mix and match each layer to make your own themes.

Styling

Styling system powered by react-style, with a declarative and pure way of adding styles. Use the power of JS, along with our easy interface to style components.

Animations

Animations handled through React props. Composable, consistent, and JS powered. Compose multiple animations per component, all within React.

Icons

We've imported a huge set of SVG icons to use with the Icon component. Or load your favorite pack of icon fonts.

View Lists

Heavy work went into building the view system. Build custom views with our ViewListMixin, nest views within each other, and all animations are handled for you.

Consistent

Everything is JS, all the way. Even the styles. With a dash of Webpack and 6to5 to make it fun.

Powerful

Powered by React for performance, composability, and component's with lifecycles.

Helpful

From start to finish, reapp includes helpers for everything from data to routing to UI.

Composable

New frameworks and build systems make you gag. reapp is modular so you pick and choose what you like.

Example

<DottedViewList>
  <View title="Hot Articles">
    <List>
      {hotArticles.map(article => article.title)}
    </List>
  </View>
  <View title="New Articles">
    <List>
      {newArticles.map(article => article.title)}
    </List>
  </View>
</DottedViewList>

Views represent a single panel in an app. A ViewList can be used as a parent for Views. For now, re/app has two types of ViewLists, NestedViewList and DottedViewList but you can build your own using the ViewListMixin.

A View optionally takes a title property. This will insert a TitleBar for you automatically.

A List is a component used for, well, lists. Use a ListItem for each child, or if you pass an array, it will wrap them for you, as it does here.

Why re/app?

A new stack, designed for software instead of documents, is needed.

re/app is a collection of modules for building hybrid mobile apps using the latest JS tech.

After years of hand-rolled frameworks assisted by jQuery, there was Backbone. It began a move toward MV* on the client side, followed by Angular, Ember and many others. With full-featured frameworks we found ourselves productive, but at the cost of steep learning curves, platform tie-in, and slow performance.

Now, we have a few new things. Build tools have evolved from Grunt to Gulp/Broccolli, to Webpack. npm has grown, to become the largest package repository of any language. React and other virtual DOM libraries were released, realizing huge performance gains (and more importantly, in the case of React, the idea of components and a lifecycle).

Now we can built truly new classes of apps. They can be rendered fully by the server, and passed over with their data to the client. Virtual DOM lets web apps run up to 10x faster, for nearly native feeling. And with modern build systems, you can make your dev environment nice with transpilers, loaders, source maps, and more.

re/app is simply an early exploration into all of these new-generation technologies, by a passionate web developer with too much time. It's meant to be taken as a wish for what web tech can be in 2015, and a start on a new and modular set of helpful hybrid app modules.
