What is it?

Reapp started as an mobile UI kit for React.

Then, came the Hacker News app.
But, building that took way too long.

Not just learning React, JSX, Flux and Webpack. But finding and testing dozens of modules for routing, data, styles, requests, promises, etc.

In time, the core of that app was extracted to modules. Then, an example repo. Finally, a CLI, just to get you going quickly.

The result? Not a framework.

Reapp is a UI kit, modules, and a CLI.

Example


  export default React.createClass({
  render() {
    var { hotArticles, newArticles } = this.props;

    return (
      <DottedViewList>
        <View title="Hot Articles">
          <List>
            {hotArticles.map(article => article.title)}
          </List>
        </View>

        <View title="New Articles">
          <List>
            {newArticles.map(article => article.title)}
          </List>
        </View>
      </DottedViewList>
    );
  }
});

View Lists

A <ViewList> can be used to handle <Views>. For now, reapp has NestedViewList and DottedViewList but you can create custom ones with ViewListMixin.

A View takes children and an optional a title property.

A List is a component used for lists. Use ListItem for children, or if you pass an array it will wrap them for you automatically.

UI Features

Animations

Composable, interactive, JS powered. Use multiple animations per component, dynamically changeable through React props.

Themes

Themes are composed of three layers: constants, styles, animations. Mix and match all of them to make your own themes.

Styles

Powered by JS and react-style, our styles interface is consistent, pure, and performant, and dynamically changeable at runtime.

Views

Display lists of views with custom styles and animations using the ViewListMixin, mix, match and nest them.

What else?

We've built a set of small, optional modules that help you build apps quickly. Once generated, you can mix and match them as you like.

reapp-component

Easily decorate react classes, and inject variables to your whole stack.

reapp-pack

A generator of webpack configs, takes options and builds configs for reapp apps.

reapp-platform

Init's touch events, and gives a small set of mixins and helpers common to React apps.

reapp-reducer

A tiny reducer for use with flux apps.

reapp-request

Superagent + bluebird + promises + caching.

reapp-routes

Simple DSL for routes. We have a pre-built one for react-router. DRY route to directory requires.

reapp-server

An express and webpack server that take a variety of options, to bootstrap building reapp apps.

reapp-ui

All the components and helper mixins to create complete UI's.

From spaghetti, to jQuery, Backbone, and the MV Stars, Javascript is moving fast and improving rapidly. We have some amazing frameworks, but the cost of learning them is often high.

Amazing build tools have evolved, Webpack and 6to5 enable hassle-free development. Want ES7, 6, JSX, Flow? It's all there. And with npm we have a mature selection of fast modules.

React brings much needed performance to the DOM, and just as imporantly, components with consistent lifecycles. Not to mention Flux and immutable data.

We want the flexibility of the bazaar with the comfort and ease of a cathedral. Reapp is an exploration into building modular, beautiful apps.
