UI Kit

reapp-ui composes the bulk of reapp. It's a UI kit that gives you the pieces you need to build beautiful app UI's.

Themeing

Powerful themeing system. Themes are composed of three layers: constants, styles, animations. Mix and match each layer to make your own themes.

Styling

Styling system powered by react-style, with a declarative and pure way of adding styles. Use the power of JS, along with our easy interface to style components.

Animations

Animations handled through React props. Composable, consistent, and JS powered. Compose multiple animations per component, all within React.

Icons

We've imported a huge set of SVG icons to use with the Icon component. Or load your favorite pack of icon fonts.

View Lists

Heavy work went into building the view system. Build custom views with our ViewListMixin, nest views within each other, and all animations are handled for you.

Example

<DottedViewList>
  <View title="Hot Articles">
    <List>
      {hotArticles.map(article => article.title)}
    </List>
  </View>
  <View title="New Articles">
    <List>
      {newArticles.map(article => article.title)}
    </List>
  </View>
</DottedViewList>

A ViewList can be used to handle multiple Views. For now, reapp has NestedViewList and DottedViewList but you can create custom ones with ViewListMixin.

A View takes children, and an optional a title property. This will insert TitleBar for you automatically.

A List is a component used for lists. Use ListItem for children, or if you pass an array it will wrap them for you automatically.

Why reapp?

A new stack, designed for software instead of documents, is needed.
@lkrubner
having everybody in need of transportation buy the necessary spareparts to build a car is insane.
@jballanc

After years of spaghetti, jQuery, Backbone, and the MV Stars, we find ourselves in 2015 with slow frameworks that give you the kitchen sink with a steep learning curve.

But we have new things. Amazing build tools have evolved. From Grunt to Gulp to Brocolli to Webpack. Our npm has matured nicely, the largest package repository of any language, and we no longer need Bower or anything else.

The Virtual DOM was invented by React, we realized huge performance gains. But just as importantly, we gained composable components, with lifecycles.

We fight about modularity. We want the flexibility of the bazaar with the comfort and ease of a cathedral. We want initial renders from the server. We want to build cross platform apps, without dealing with all the ugly of JavaScript.

reapp is an early exploration into making all of this reality.
