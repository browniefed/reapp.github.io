What is it?

A CLI that bootstraps your app, our UI kit, and some modules to help.

We've used it to build a Hacker News app and extracted what we learned into Reapp.

It's not a framework. Just modules, and a CLI.

Example


import React from 'react';
import { DottedViewList, View, List } from 'reapp-ui/all';

export default React.createClass({
  render() {
    return (
      <DottedViewList>
        <View title="Hot Articles">
          <List>
            {HotArticles.map(a => a.title)}
          </List>
        </View>

        <View title="Other">
          <p>Second view contents</p>
        </View>
      </DottedViewList>
    )
  }
});

DottedViewLists show Views side-by-side, like the Twitter app.

The ViewListMixin can be used to create ViewLists. For now, reapp has NestedViewList and DottedViewList.

Pass a title property to a View to automatically insert a TitleBar.

Similarly, when passed an array of items, a List will automatically wrap each in a ListItem for you.

The UI

Animations

Interactive, customizable, composable JavaScript animations. Passed through props, built for React.

Styles

JavaScript styles with react-style for composable, dynamic, styles (+ a declarative interface).

Themes

Set constants, styles, animations to make custom themes. Change a few colors or swap the whole theme out!

Views

Variety of views, including View Lists, and even nested View Lists that support custom styles & animations.

Components

Include components individually without adding bulk to your app. Including a set of SVG icons for crisp rendering.

Mixins

All the animation and styling handlers, as well as variety of helpers are usable outside the library with our mixins.

What else?

We've built a set of small, optional packages that help you build apps quickly. Once generated, you can mix and match them as you like.

reapp-component

Easily decorate react classes, and inject variables to your whole stack.

reapp-pack

Generates webpack configs & run a webpack dev server with hot reloading.

reapp-platform

Init's touch events, and gives a small set of mixins and helpers common to React apps.

reapp-reducer

A tiny reducer for use with flux apps.

reapp-request

Superagent + bluebird + promises + caching.

reapp-routes

Simple DSL for routes. We have a pre-built one for react-router. DRY route to directory requires.

reapp-server

An express and webpack server that take a variety of options, to bootstrap building reapp apps.

From spaghetti, to jQuery, Backbone, and the MV Stars, Javascript is moving fast and improving rapidly. We have some amazing frameworks, but the cost of learning them is often high, and lock-in sucks.

We have amazing build tools, Webpack and 6to5 enable hassle-free development, and react-hot-loader makes developing incredibly fast. Get ES7, ES6, JSX, Flow, and more. And npm is all you need to pull it all together.

Reapp avoids the framework trouble by not being one. If you like React, you can enhance it with Reapp.

Enjoy React's performance, components with consistent lifecycles, mixins, and more, without having to figure out the routing, data, and many other small details.
